# 43_PROJECT_KICKOFF_SYSTEM — Project Initialization Framework

## 1. Why This Exists & Why It Works
Project misalignments usually start during the first week. If stakeholder expectations, visual directions, and metrics are not aligned early, layout creation suffers. This document establishes guidelines for project kickoff and discovery sessions. It works by setting clear project parameters and aligning expectations early.

---

## 2. Discovery Workshop Framework
Conduct three workshops with key client stakeholders to gather project requirements:
1. **The Visual Direction Workshop**: Review the 11 creative directions to align on a target style.
2. **The Business Alignment Workshop**: Detail the product's primary competitor and unique differentiator.
3. **The KPI Target Workshop**: Define measurable success metrics for the launch.

---

## 3. Kickoff Discovery Templates

### Template: Stakeholder Interview Guide
Ask stakeholders these key questions to define project constraints:
* *"If this digital platform was a physical space, what materials, lighting, and textures would define it?"*
* *"Who is our primary visual opponent, and why is their current layout failing?"*
* *"What is the single action we want a user to take within 30 seconds of landing?"*

### Template: Goal Definition Worksheet
```markdown
### Project Parameters

1. **Target Archetype Alignment**: [e.g. Sovereign, Sage, Creator]
2. **Key Competitor Visuals to Avoid**: [Specify styles]
3. **Primary Action Metric Target**: [e.g. Demo requests, lead registrations]
4. **Hardware Baseline Performance Target**: [e.g. SpeedIndex < 1.5s on mid-tier mobile]
```

---

## 4. When to Use & Avoid
* **When to Use**: Project week 1, initial client kickoff calls, creative alignment workshops.
* **When to Avoid**: Coding setup, asset compilation, or styling details.
