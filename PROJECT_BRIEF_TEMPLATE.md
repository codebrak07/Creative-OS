# Creative Operating System (COS) — Project Brief Template

To initialize a new project and configure the retrieval of relevant COS manuals, populate the 8 input parameters below.

```markdown
# PROJECT BRIEF

1. **Industry**: 
   * [Define the vertical, e.g. B2B AI Infrastructure, Ultra-Luxury Watches, Editorial Travel]
   * *Rationale*: Influences compliance, user interface density, and standard user patterns.

2. **Audience**: 
   * [Detail user segments, demographics, and technical proficiency, e.g. Senior Kernel Engineers]
   * *Rationale*: Dictates typography sizes, visual weight, reading speeds, and layout layout systems.

3. **Business Goal**: 
   * [Identify the primary conversion goal, e.g. Establish category authority to lift average order value (AOV)]
   * *Rationale*: Anchors strategic priorities and KPI targets for the design and content layers.

4. **Brand Personality**: 
   * [Define tone, e.g. Hyper-engineered & stoic, warm editorial, premium minimalist]
   * *Rationale*: Directs fluid font choices, visual grids, color harmonies, and layout spacing.

5. **Competitive Landscape**: 
   * [List 3-4 key competitors and specify their visual styles to avoid imitation]
   * *Rationale*: Ensures the design breaks standard market visual patterns and creates differentiation.

6. **Primary Differentiator**: 
   * [Specify the technological, brand, or operational moat of the product]
   * *Rationale*: Serves as the central storytelling thread and dictates the primary hero landing experience.

7. **Desired Creative Direction**: 
   * [Choose from the 11 directions defined in 22_CREATIVE_DIRECTION_GENERATOR.md]
   * *Rationale*: Configures the default layout grids, color sets, and interaction presets.

8. **Success Metrics**: 
   * [Define conversion lift rates, Awwwards targets, or performance budgets, e.g. SpeedIndex < 1.2s]
   * *Rationale*: Feeds the Quality QA and Review Audit checks for project release approval.
```
